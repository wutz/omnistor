# DASE 架构（Disaggregated Shared-Everything）

DASE = Disaggregated Shared-Everything（解耦共享一切），是 Vastdata vastbox 的核心架构思想。OmniStor 沿用此范式。

## 与传统架构对比

| 架构 | 计算与存储 | 共享 | 扩展瓶颈 |
| --- | --- | --- | --- |
| 传统 Scale-up | 耦合 | 否 | 单点 |
| Scale-out (存算一体) | 耦合 | 部分 | 节点间副本放大 |
| **DASE (OmniStor)** | **解耦** | **全部共享** | **近乎线性** |

## 解耦（Disaggregated）

- **计算前端**（NFS / S3 / iSCSI 网关）无状态，可独立横向扩展，按协议负载弹性伸缩。
- **元数据服务**与**数据服务**分离部署，各自按负载扩展（元数据偏 IOPS，数据偏带宽与容量）。

## 共享一切（Shared-Everything）

- 任意数据节点都可服务任意对象的读写请求——对象副本全局可见，不存在"主副本绑定节点"。
- 写入路径：前端 → 元数据定位 → 任一数据节点接收 → 该节点协调跨节点副本/纠删写入。
- 读取路径：前端 → 元数据定位 → 选择最近/最闲的数据节点读取。
- 好处：负载天然均衡，故障切换无数据迁移，容量与带宽随节点数线性增长。

## 一致性模型

- 元数据：强一致，RAFT 多副本写入。
- 数据：副本间通过 quorum 写保证一致性；读取优先 quorum-read 或主副本。
- 跨介质分层迁移时，迁移完成前旧副本保留，保证读不中断。

## 与硬件的协同

DASE 配合 ebox 硬件方案：每个 ebox 是一个标准化的"共享存储单元"，多个 ebox 组成集群，软件层把它们视为统一的共享池。详见 [hardware.md](hardware.md)。
