# 集群拓扑与容量规模

## 规模目标

- 对象数量：**万亿级（10¹²）**
- 单集群容量：**1 EiB（2⁶⁰ 字节）**

## 容量推演（示例）

以配置 B（TLC SSD + HDD）估算，假设单 HDD = 24 TB：

| 项目 | 数值 |
| --- | --- |
| 单 ebox HDD 数量 | 90 |
| 单 ebox 原始容量 | 90 × 24 TB = 2,160 TB ≈ 2.1 PiB |
| 纠删开销 (k+m, 如 8+3) | 有效系数 ≈ 0.73 |
| 单 ebox 有效容量 | ≈ 1.5 PiB |
| 达到 1 EiB 所需 ebox | 1024 PiB / 1.5 ≈ **683 个 ebox** |

> 以上为量级估算，实际取决于纠删策略、副本数与介质选型。

## 元数据规模

万亿对象意味着元数据存储压力极大：

- 单对象元数据若 ~1 KB，万亿对象 ≈ 1 EiB 元数据量级——必须分片。
- 元数据服务按**哈希分片 + RAFT 复制**横向扩展，单分片承载 ~十亿级对象。
- 目录树与索引采用 KV 存储（基于 SCM SSD 低延迟介质）。

## 拓扑示意

```
Cluster (1 EiB, ~700 eboxes)
├── ebox-001 .. ebox-700
│   ├── Frontend gateways (NFS/S3/iSCSI)
│   ├── Metadata shards (RAFT groups)
│   └── Data nodes (QLC/HDD pools)
└── Global coordinator (集群编排、放置决策、监控)
```

## 扩展性设计

- **前端**：无状态，按 QPS 弹性扩。
- **元数据**：分片再平衡，新增分片不打断服务。
- **数据**：新 ebox 加入即纳入共享池，放置引擎自动在新节点上分配新写入。

详见 [scaling.md](../scaling.md)。
