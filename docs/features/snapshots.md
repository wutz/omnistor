# 快照与克隆

OmniStor 提供**元数据级 COW（copy-on-write）快照**：创建瞬时完成、与数据量无关；快照可派生**可写克隆**；快照可**导出到对象存储**（snap-to-object）用于备份、跨集群复制与灾备恢复。

## 模型

快照作用于一个**命名空间范围**（文件系统目录树 / 对象桶 / 块卷），本质是该范围元数据的一个**只读世代（generation）视图**：

```
gen 0 ──── gen 1 ──── gen 2 ──── live（可写头）
  │           │
snap:daily  snap:pre-upgrade
                │
                └── clone "test-env"（可写头，分叉）
```

- **元数据 COW**：Bucket 内每个键的值带世代号。创建快照 = 把范围的"当前世代"冻结并递增可写头的世代号——**一次元数据操作，瞬时完成，与对象数量无关**。
- **数据零拷贝**：快照不复制任何数据 extent。写新位置的纠删设计（见 [data-protection.md](../architecture/data-protection.md)）意味着覆盖写天然产生新 extent，旧 extent 被快照世代引用则不回收。
- **增量演进**：相邻世代只存差异（新写/覆盖/删除的键），空间开销 = 变更量。
- 快照数量上限高（每范围数千），支持定时策略（每小时/每天）与保留规则。

## 读快照

- 读快照视图 = 按 `(key, generation ≤ snap_gen)` 取最新版本——B-tree 有序性使其只是一次带世代过滤的查找。
- 快照读与 live 读走同一条路径、同一套 QoS；快照不是"另一份数据"，只是另一个入口世代。

## 可写克隆

- 从任意快照**分叉出新的可写头**：克隆与源共享分叉点之前的全部元数据与数据，分叉后各写各的世代链。
- 典型用途：开发/测试环境秒级拉起生产数据副本、块卷模板批量派生、AI 实验数据集版本化。
- 克隆记账：共享部分不重复计配额，分叉后的新写计入克隆所属租户/范围的配额（见 [quota.md](quota.md)）。

## 删除与回收

- 删除快照 = 移除该世代的引用；某 extent 不再被任何世代（含 live 与克隆）引用时，由后台回收（低优先级 QoS）。
- 引用计数按 extent 维护在 Bucket 内，与指针切换同一事务域，无独立 GC 扫描。

## Snap-to-Object（导出到对象存储）

把快照**完整物化**到外部对象存储（S3）：

- 导出内容 = 该世代的元数据序列化 + 所有被引用的数据 extent（自包含格式：数据 + 校验 + 加密，与分层外溢格式一致，见 [tiering.md](tiering.md)）。
- **增量导出**：已导出过前代快照时，只上传世代差异。
- 用途：
  - **备份**：集群外的独立副本，介质故障域完全隔离；
  - **跨集群复制/迁移**：另一个 OmniStor 集群从对象存储**导入**快照，重建出完整命名空间；
  - **灾备**：主集群全损后从对象存储恢复到任意新集群。
- 与分层（tiering）的区别：分层是 live 数据的容量外溢，元数据留在本集群；snap-to-object 是自包含的完整导出，不依赖源集群存活。

## 一致性语义

- 快照点是**范围内的崩溃一致点**：创建时短暂屏障（flush 在途写 + 冻结世代），保证快照内不出现"半个事务"。
- 块卷快照可与主机侧配合（fsfreeze / VSS）做到应用一致。
- 跨范围（多个桶/卷）的一致组快照：同一屏障内冻结多个范围，作为一个恢复单元。

## 与多租户/QoS 的协同

- 快照/克隆操作是租户内自助操作（租户管理员权限，见 [multitenancy.md](multitenancy.md)）。
- 快照占用的空间计入租户配额（差异量），防止"快照无限堆积绕过配额"。
- 导出/导入流量走后台低优先级令牌 + 外联带宽上限。
