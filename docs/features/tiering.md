# 介质分层与数据放置

OmniStor 的 Data 层包含多级介质，放置引擎按数据温度在介质间迁移。

## 介质层级

```
热 → 冷
SCM SSD ──> TLC SSD ──> QLC SSD ──> HDD
(元数据/缓存)  (热数据)    (温/主力)   (冷/归档)
```

三种支持的硬件配置对应不同的可用层级：

| 配置 | 上层（热） | 下层（冷） |
| --- | --- | --- |
| SCM + QLC | SCM SSD | QLC SSD |
| 纯 TLC | TLC SSD | —（单介质，仅分区内整理） |
| TLC + HDD | TLC SSD | HDD |

## 放置策略

- **新写入**：先落上层（SCM/TLC）作为写缓存，异步下沉到下层（QLC/HDD）。纯 TLC 配置无下层，写缓存与主数据同介质，不跨介质下沉。
- **读命中**：上层命中直接返回；未命中从下层读，并按策略决定是否提升（promotion）。
- **温度判定**：基于访问频度（LFU 窗口）与时间衰减，后台扫描周期性迁移。
- **大对象直写**：超阈值的大对象可绕过缓存直写下层，避免缓存污染。

## 迁移一致性

- 迁移期间旧副本保留，读不中断。
- 迁移完成后原子切换元数据指针，再异步回收旧副本。
- 迁移失败可回滚，不影响数据可用性。

## 与 QoS/Quota 的协同

- 分层迁移不改变对象归属，配额计数不变。
- 迁移本身消耗后台带宽，受全局后台 QoS 约束，避免影响前台。
