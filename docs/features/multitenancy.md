# 多租户（Multi-Tenancy）

OmniStor 在一套集群上支持多个相互隔离的**租户（Tenant）**：每个租户拥有独立的命名空间视图、访问入口、认证体系、QoS/Quota 配置与加密密钥，像使用独立集群一样使用共享集群的一个切片。

## 租户模型

```
Cluster
├── Tenant "acme"
│   ├── 命名空间: /tenants/acme/**（文件） · 桶集合（对象） · 卷集合（块）
│   ├── 访问入口: VIP 池 10.1.8.0/28 (VLAN 108)
│   ├── 认证: acme 自有 LDAP / OIDC
│   ├── QoS / Quota / 加密密钥 / 租户管理员
├── Tenant "globex"
│   └── …（完全独立的一套）
└── 集群管理员（跨租户运维，可被禁止读取租户数据）
```

- **租户是硬边界**：命名空间、认证、密钥互不可见；桶名、用户名、卷名只需租户内唯一。
- 一切资源（文件系统子树、S3 桶、块卷）**创建时归属唯一租户**，不可跨租户共享；跨租户数据流动只能走显式的复制/拷贝。

## 命名空间隔离

| 协议 | 隔离方式 |
| --- | --- |
| 文件（NFS） | 每租户一个或多个**独立导出树**，租户内可再建多个导出；跨租户路径互不可见 |
| 对象（S3） | 每租户独立的**桶命名空间**与访问密钥体系，桶名租户内唯一即可 |
| 块（iSCSI/NVMe-oF） | 卷与 target 按租户划分，租户只能发现自己的 target |

实现上复用元数据 Bucket 模型：租户 ID 是所有元数据键（inode/桶/对象 key/卷）的**最高位前缀**，天然随哈希散列到全部 Bucket——租户隔离不引入新的元数据层，一个 Bucket 同时服务多个租户的分片，靠键前缀严格隔离。

## 访问入口与网络隔离

- **每租户一个 VIP 池**：租户客户端只连接本租户的虚拟 IP 组；VIP 在 CNode 间浮动实现负载均衡与故障切换。
- **VLAN / 子网绑定**：VIP 池可绑定 VLAN，租户流量在网络层隔离，支持租户网络地址重叠（不同租户可用相同网段）。
- CNode 是共享的：同一 CNode 可同时服务多个租户的 VIP，协议栈按 VIP → 租户上下文路由；也可将 CNode 分组**专属**给大租户（物理隔离前端）。

## 认证与访问控制

- **每租户独立的身份源**：各租户可对接自己的 LDAP / AD / OIDC，或使用本地用户库；身份源之间互不可见。
- **S3 密钥按租户签发**：access key 绑定租户上下文，跨租户的桶访问在认证层直接拒绝。
- **两级管理**：
  - 集群管理员：建删租户、分配容量/QoS 上限、运维硬件——可配置为**无权读取租户数据**（管理与数据平面分离）。
  - 租户管理员：在租户配额内自助管理导出/桶/卷、用户与 ACL、子配额与子 QoS，无需集群管理员介入。

## QoS 与 Quota：租户为第一级

复用既有机制（见 [qos.md](qos.md)、[quota.md](quota.md)），租户是最外层的记账与限速单位：

- **两级 QoS**：集群管理员给租户设总上限（Metadata IOPS / Data IOPS / Data BW），租户管理员在上限内给自己的桶/卷/导出细分——两级都用分片令牌桶，天然叠加。
- **两级 Quota**：租户总容量/对象数由集群管理员设定；租户内再分桶/目录/卷子配额。
- **性能隔离**：租户令牌桶 + Bucket 入口反压，防止单租户打满共享 CNode；机制上限速在入口执行，噪声租户的过载请求不会进入共享队列。

## 数据隔离与加密

- **逻辑隔离（默认）**：所有租户共享存储池与纠删条带，靠键前缀 + 认证隔离——成本最优，池间均衡自由度最大。
- **物理隔离（可选）**：存储策略可将租户绑定到专属存储池（见 [pools.md](../architecture/pools.md)）——数据条带不与其他租户混布，满足合规要求；代价是失去共享池的统计复用。
- **每租户加密密钥**：静态加密按租户使用独立密钥（信封加密：租户 KEK → extent DEK），支持对接租户自管 KMS；删除租户时销毁 KEK 即完成密码学擦除（crypto-shredding）。

## 计量与可观测

- 按租户导出用量（容量、对象数）与性能（IOPS/BW/延迟分位）指标，供计费与容量规划。
- 审计日志按租户隔离：租户管理员只能看到本租户的操作记录。

## 配置示例（概念）

```yaml
tenant:
  name: "acme"
  vip_pool: { subnet: "10.1.8.0/28", vlan: 108 }
  auth: { type: oidc, issuer: "https://sso.acme.example" }
  qos: { metadata_iops: 50000, data_iops: 100000, data_bw: "10GiB/s" }
  quota: { capacity: "1PiB", object_count: 10000000000 }
  encryption: { kms: "https://kms.acme.example", key_id: "acme-kek-1" }
  placement: shared            # shared | dedicated-pool:<pool-name>
  admins: ["alice@acme"]
```
